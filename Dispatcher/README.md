## О проекте

При аварийных отключениях линий электропередачи диспетчеру приходится одновременно учитывать множество факторов: погоду, особенности местности, состояние бригад. Готового простого инструмента, который объединяет все эти данные и даёт рекомендации, не существовало.

Данный проект — это программный помощник, который:

- Собирает реальные данные о погоде из Open-Meteo
- Получает информацию о рельефе через OpenStreetMap
- Рассчитывает уровень риска для каждого района
- Показывает цветовую карту в веб-интерфейсе
- Выдаёт готовые рекомендации через чат-бота ВКонтакте
- Использует нейросеть g4f для генерации практических советов

Система реализована на примере Вологодской области, но может быть легко адаптирована для любого региона.

## Возможности

- 🗺️ Интерактивная карта с цветовой индикацией уровней риска

- 🤖 Генерирует рекомендации через нейросеть g4f
- ⚙️ Имеет резервный алгоритм при недоступности ИИ

### Серверная часть

- 📡 Запросы к Open-Meteo API — погода в реальном времени
- 🗂️ Геокодирование через Nominatim — определение типа местности
- 📐 Расчёт уровня риска по 5 факторам с весовыми коэффициентами
- 🔄 Резервные демо-данные при отказе внешних сервисов
- 📝 Логирование всех операций для диагностики

## Установка и запуск

### Требования

- Python 3.9 или выше
- pip (менеджер пакетов)
- Токен группы ВКонтакте (только для чат-бота)

### 1. Клонирование репозитория

```bash
git clone https://github.com/ArtemMac00/Portfolio.git
cd Portfolio/Dispatcher
```

### 2. Установка зависимостей

```bash
pip install flask flask-cors requests vk_api aiohttp
```

Для работы нейросети (опционально):

```bash
pip install g4f
```

Бот будет работать и без g4f — в этом случае включится локальный анализатор на основе правил.

### 3. Запуск сервера

```bash
python backend.py
```

Сервер запустится на `http://localhost:5000`. Веб-интерфейс откроется автоматически при переходе по этому адресу.

### 4. Запуск чат-бота ВКонтакте

1. Создайте группу ВКонтакте
2. В настройках группы включите «Сообщения сообщества» и «Чат-бот»
3. В разделе «Работа с API» создайте ключ
4. Откройте файл `bot.py` и укажите токен и ID группы:

```python
VK_TOKEN = "ваш_токен_группы"
GROUP_ID = ваш_id_группы
```

5. Запустите бота:

```bash
python bot.py
```

Бот начнёт слушать сообщения и отвечать. При первом запуске в консоли появится информация о статусе подключения.

## Как работает расчёт риска

Уровень риска вычисляется по формуле, учитывающей пять факторов:

| Фактор | Вес | Что учитывается |
|--------|-----|-----------------|
| Ветер | 0.3 | Скорость ветра, порывы |
| Температура | 0.2 | Мороз, риск обледенения |
| Осадки | 0.2 | Дождь, снег, нагрузка на провода |
| Обледенение | 0.2 | Сочетание влажности и отрицательной температуры |
| Погодные явления | 0.1 | Гроза, туман, метель |

Каждый фактор даёт от 0 до 1 балла по пороговым значениям. Балл умножается на вес. Сумма всех взвешенных баллов умножается на 10. Итог — число от 0 до 10.

Пороговые значения:

| Фактор | Низкий | Средний | Высокий | Критический |
|--------|--------|---------|---------|-------------|
| Ветер (м/с) | < 5 | 5–10 | 10–15 | > 15 |
| Температура (°C) | > 0 | 0...-10 | -10...-20 | < -20 |
| Осадки (мм/ч) | < 2.5 | 2.5–7.5 | 7.5–15 | > 15 |

Дополнительно учитывается тип местности: лес (повышает риск падения деревьев), холмы (усиленная ветровая нагрузка), приозёрные зоны (туманы, коррозия).

## Надёжность

В системе предусмотрены механизмы устойчивости к сбоям:

- При недоступности Open-Meteo используются резервные демо-данные, и интерфейс продолжает работать
- При отказе нейросети g4f автоматически включается локальный анализатор на основе правил — пользователь всегда получает ответ
- Все ошибки записываются в лог с указанием времени и причины
- Внешние запросы имеют таймаут 10 секунд, чтобы не зависать при проблемах с сетью
